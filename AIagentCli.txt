AI Agent CLI 设计文档
1. 项目概述
1.1 项目目标
设计并实现一款高性能、模块化的 AI Agent CLI 工具，借鉴 Claude Code 泄露源码中的核心设计思想，支持多种 LLM 后端（本地模型 + 云 API），提供交互式终端体验，具备代码理解、文件操作、工具调用等完善功能。

Claude Code 的泄露源码（约 1900 个 TypeScript 文件，51.2 万行代码）展示了非常成熟的生产级 AI agent harness 架构，包括 REPL 启动、QueryEngine、工具注册、Slash 命令、权限系统和多层状态管理等。这些设计模式值得在新工具中借鉴并移植到 Rust 生态。

1.2 核心理念
理念	说明
Rust Native	原生性能、内存安全，用 Rust 替代原版 TypeScript/Bun/React Ink 技术栈
模型中立	支持本地模型（Ollama、GGUF）和云 API（OpenAI、Anthropic、DeepSeek、Gemini 等）的自由切换
模块化	核心与工具解耦，token 计费、日志、加密等以中间件形式注入
隐私优先	本地模型零数据离境，用户完全控制数据流向
渐进增强	基础功能可在纯本地运行，高级功能（如多模型编排）可选使用云服务

2.2 借鉴 Claude Code 的五种核心设计模式
根据对泄露源码的分析，以下五种 Agent 设计模式尤其值得在 Rust 实现中借鉴：

模式	说明	Rust 实现方案
QueryEngine + 工具注册	自然语言输入经 QueryEngine 解析，匹配并调用注册的工具	基于 trait 的工具注册表，宏系统简化注册过程

上下文压缩与记忆管理	会话压缩和长期记忆存储，KAIROS 后台守护机制包含"dream"内存整理	实现滑动窗口 + 摘要压缩的双层机制
2.3 技术选型优势
Rust 原生替代原版 Claude Code 的 TypeScript/Bun/React Ink 技术栈，带来以下优势：

性能：零成本抽象，启动速度和内存占用远超 Node.js 运行时

并发：tokio 异步运行时，原生支持 agent loop 中的并发工具调用

类型安全：编译时杜绝运行时类型错误，提升 agent 系统的健壮性

生态工具：Ratatui 用于 TUI、candle 用于原生推理，均为成熟的 Rust 生态选择

4. 核心系统规范
4.1 Agent Loop
Agent Loop 是系统的核心执行引擎。模型收到用户请求，确定需要执行的操作并生成工具调用请求，模型接收工具执行结果后反复选择并调用下一个工具，直到 Agent 确定问题已解决并返回最终结果。

工作步骤：

解析用户 Prompt 转换为 Agent 任务

构建系统提示和消息上下文

发送给 LLM 推理，响应包含工具调用请求

执行工具（权限检查后）返回结果

将结果附加到上下文继续调用 LLM

重复 3-5 直至出现 end_turn 或达到最大轮次

返回最终输出

循环终止条件包括：end_turn（正常完成）、tool_use（等待工具调用）、max_tokens（超限）和 refusal（安全拒绝）。

4.2 工具系统
参考泄露源码中的设计，采用三层工具架构：

内置工具：Read/Write/Edit/Run/Grep/Glob 等文件操作和文本搜索工具

权限系统：按工具类型和路径精细控制执行权限，危险操作需用户确认

计划工具：作为上下文工程策略，不执行实际操作，仅用于让模型输出结构化计划。这种看似"无用"的设计，实际上对复杂任务长期执行时保持 Agent 方向感非常关键

工具注册使用 Trait 系统：
pub trait Tool: Send + Sync {
    fn name(&self) -> &str;
    fn description(&self) -> &str;
    fn parameters(&self) -> serde_json::Value;  // JSON Schema
    fn execute(&self, params: serde_json::Value) -> impl Future<Output = ToolResult>;
}

4.4 记忆与上下文管理
借鉴 Claude Code 的多层状态管理，实现三层记忆架构：

层级	内容	实现方式
工作记忆	当前会话消息列表	Vec<Message> 滑动窗口
短期记忆	工具调用结果、代码片段	摘要压缩 + 关键信息提取
长期记忆	项目知识、历史决策	向量数据库 + RAG 检索
上下文窗口压缩策略：

当 token 计数超过阈值，触发压缩

保留最近 N 轮完整对话

将更早对话生成摘要，作为系统提示附加

4.5 子代理系统
借鉴 Claude Code 的子代理设计，可将复杂任务拆解给专用子代理：

主代理分析任务复杂度

如需拆分，创建隔离的子代理实例

子代理拥有独立上下文，可调用受限工具集

子代理完成后返回结果给主代理

主代理整合结果继续执行

子代理与主代理上下文隔离，避免互相污染，同时允许并行执行多个独立子任务。

    ▼

    │
    ▼

    │
    ▼

6. 多模型编排
借鉴 CogniRepo 和 CAMP 的多模型协同思想，实现查询复杂度 → 构建上下文 → 路由到最佳模型的自动编

pub struct ModelRouter {
    config: RouterConfig,
}

impl ModelRouter {
    pub async fn route(&self, query: &str) -> ModelSelection {
        // 1. 分析查询复杂度
        let complexity = self.analyze_complexity(query);
        
        // 2. 根据复杂度选择模型
        let selected = match complexity {
            Complexity::Simple => ModelType::Fast,      // 本地小模型
            Complexity::Complex => ModelType::Reasoner, // Claude/DeepSeek
            Complexity::Code => ModelType::CodeGen,     // CodeLLama
        };
        
        // 3. 构建上下文并路由
        self.build_context(query, selected).await;
        selected
    }
}

路由策略
查询类型	推荐模型	说明
简单解释	本地小模型（Ollama）	快速响应，低成本
深度推理	Claude / DeepSeek R1	复杂逻辑推理
代码生成	CodeLLama / GPT-4	专注代码任务
代码检索	本地 Embedding + BM25	不经过 LLM

10. 安全与权限机制
参考泄露代码中发现的权限系统，设计三层防护：

层级	措施	说明
工具白名单	allowed_tools 配置	默认禁用危险操作（网络、系统命令等）
实时确认	高影响操作时触发权限请求	用户决定是否放行
成本保护	Token 计数和预算限制	设置单次会话/API 调用的最大 token 上限
沙盒环境	容器化为嵌套选项	高风险任务降级至 Docker 隔离执行
此外，可参考泄露源码中的防蒸馏设计——对于云 API 模式提供可选的 fake_tools 注入机制防止训练数据被盗用。API Key 采用本地 AES 加密存储，日志中自动脱敏敏感信息。

11. 可观测性与开发体验
设计三个层面的可观测性：

请求层：API 请求/响应 tracing，问题定位时可以精确到每一次 HTTP 通信

Agent 层：Agent 循环内嵌 log_plan 和 log_action，可将 Planning 的决策序列渲染为结构化视图

工具层：工具执行挂钩 OpenTelemetry，方便对接 Prometheus 进行长时间运行的分析

用户体验方面，借鉴 Claude Code 经典设计提供以下辅助功能：

交互式确认：高影响文件写入、删除操作时，在 TUI 底部展示 diff 差异和权限确认对话框

检查点恢复：长时间 Agent 任务支持中断保存，下次执行可加载进度

Token 计数器：在状态栏实时展示本轮对话 Token 消耗

内嵌命令：快速切换模型、切换模式（高效推理 ↔ 深度思考）

这套设计完整覆盖了从本地模型推理到云端 API、从简单对话到复杂 Agent 编排的多种场景，可灵活满足不同开发环境和隐私需求。
